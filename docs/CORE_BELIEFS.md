# 核心信念

本项目 agent-first 运行的不可违背原则。所有 agent 行为须在此框架内。

## 1. Agent 在仓库外无法访问的知识等于不存在

设计决策、架构讨论、bug 分析必须作为仓库内的版本化产物存在（markdown / yaml / code）。
Slack 讨论、口头约定、人脑中的知识不计入 agent 可用上下文。

## 2. 机械约束优于自然语言建议

能用 linter / CI / hook 强制执行的规则，就不要用 CLAUDE.md 建议。
Agent 会遵守编译错误，但可能忽略"建议"。

## 3. 纠正是廉价的，等待是昂贵的

Agent 犯错可以快速修复（分钟级），但等待人类审批是小时级阻塞。
除最终合入主分支的人类门控外，所有中间环节应 agent-to-agent 自主完成。

## 4. 人类品味编码一次，持续执行

代码风格、架构偏好、性能标准应编码为可检查的规则（linter / QUALITY_SCORE.md），
而不是每次口头告诉 agent。编码一次 → 所有后续 agent 自动遵循。

## 5. 偏好无聊的技术

选择 API 稳定、训练集中充分表征的技术。Agent 对主流技术的建模更准确。
宁可选"无聊但可靠"的方案，不选"新潮但 agent 不熟"的方案。

## 6. 渐进式披露，不是一次性灌输

上下文是稀缺资源。根 CLAUDE.md 只放指针，深层信息让 agent 按需读取。
当所有东西都"重要"，没有东西是重要的。

## 7. Scope 隔离

每个 agent 只感知它需要的模块。笛卡尔积膨胀时，隔离 > 全量感知。
